§ 01

Lifecycle Hooks

Các hook ở ranh giới session và durability boundary. Chủ yếu là non-blocking, nhưng nay đã làm nhiều hơn việc "in status": có bootstrap session-state, archival, memory hygiene và recovery contract rõ ràng.

Hook Event Mô tả
session-start.sh
 SessionStart Session bootstrap + recovery. Bơm working context ngay khi mở session và bảo đảm contract với production/session-state/active.md được khởi tạo nếu còn thiếu.
  • Current branch + 5 commits gần nhất
  • Trạng thái sprint / milestone đang chạy
  • Số open bug + TODO/FIXME trong src/
  • Recover session state cũ hoặc bootstrap active.md mới
  • Liệt kê indexed GitNexus repos nếu có
detect-gaps.sh
 SessionStart Documentation gap scanner. Chạy một health scan nhẹ lúc vào session để phát hiện code shape đã lớn hơn mức tài liệu đang có.
  • Fresh project hoặc prototype thiếu README / concept docs
  • Codebase lớn nhưng design docs còn mỏng
  • src/core / src/api tăng trưởng nhưng thiếu architecture docs
  • Gợi ý lệnh như /start hoặc /reverse-document
session-stop.sh
 Stop Archive + durability bridge. Kết thúc session bằng cách archive live state, tổng hợp log, cập nhật MEMORY.md và chỉ khi cần mới gọi memory consolidation.
  • Đọc cả agent-audit.jsonl và writes.jsonl để thống kê session chính xác hơn
  • Archive session vào .claude/memory/archive/sessions/
  • Cập nhật dòng "Last session" trong .claude/memory/MEMORY.md
  • Archive rồi xóa production/session-state/active.md
  • Trigger auto-dream.sh theo điều kiện hygiene
auto-dream.sh
 sub-process Memory hygiene utility. Không phải registered hook trực tiếp; đây là utility script được session-stop.sh gọi có điều kiện để dọn và hợp nhất memory.
  • Archive file cũ hoặc low-value memory artifacts
  • Prune broken links trong MEMORY.md
  • Tránh tạo no-op dream logs khi không có thay đổi thực sự
§ 02

Security Layer

Hooks thuộc PreToolUse. Đây là lớp prevention chính của harness: chặn lệnh nguy hiểm, kiểm tra commit / push path, và áp circuit-breaker cho Task.

⚠
Hành vi blocking: Các hook trong layer này có thể trả về exit 2 để chặn tool call. bash-guard.sh phụ thuộc vào jq và fail-closed nếu thiếu; riêng circuit-guard.sh hiện thiên về fail-open khi thiếu parser để không khóa cứng runtime.
Hook Event Mô tả
bash-guard.sh
 PreToolUse:Bash blocks Destructive command prevention. Chặn các shell pattern nguy hiểm trước khi Bash tool call chạy. Thiếu jq → block thay vì regex-parse mù.
  • Fork bombs, rm -rf variants trên /, ., ./, *
  • Ghi vào .env qua tee hoặc redirection
  • Disk formatting / raw disk write patterns
  • curl | bash, wget | sh, destructive cron deletion
  • Soft-warn cho destructive SQL, git reset --hard, git clean, volume deletion
validate-commit.sh
 PreToolUse:Bash blocks Commit-path validator. Chỉ kích hoạt khi lệnh là git commit; kiểm tra staged contents, sensitive files và chất lượng payload trước khi commit đi vào history.
  • Kiểm tra staged files và format commit message
  • Validate định dạng Conventional Commits message
  • Từ chối .env, *.key, credentials.json và JSON payload lỗi trong assets/data
  • Quét secret patterns trong diff và có thể in GitNexus blast-radius summary
  • Watchdog 25s để degrade fail-open có cảnh báo, thay vì chết im dưới outer timeout
validate-push.sh
 PreToolUse:Bash blocks Push-path screening. Theo dõi lệnh git push, cảnh báo protected branches và quét staged diff cho các secret phổ biến trước khi push.
  • Cảnh báo mạnh khi đẩy lên main, master, develop
  • Quét Anthropic, OpenAI, GitHub, Slack, AWS, Google và private key patterns
  • Kiểm tra DATABASE_URL, bearer tokens, password / secret assignments
  • Vẫn là gate cuối trước khi diff rời máy local
circuit-guard.sh
 PreToolUse:Task blocks ADR-004 read-path enforcement. Đọc state machine của circuit breaker và chặn Task khi runtime đã vào trạng thái lỗi.
  • CLOSED → pass-through bình thường
  • HALF_OPEN → cho probe traffic đi qua và phát WARN
  • OPEN → block Task cho đến khi hết TTL hoặc reset
  • Tự tạo .claude/memory/circuit-state.json nếu chưa tồn tại
  • Chỉ chặn Task, không chặn Bash / Read / Write
§ 03

Enrichment Layer

Lớp enrichment hiện không còn chỉ là memory read-path. Từ snapshot này, UserPromptSubmit đã trở thành cả read-path lẫn deterministic write-path cho Tier 2 memory.

Hook Event Mô tả
prompt-context.sh
 UserPromptSubmit Memory read-path injection. Trích keyword từ user prompt, chọn tối đa 3 memory files liên quan và inject thành additionalContext dạng data-only.
  • Sanitize instruction-like patterns như ignore previous instructions, act as, role prefixes
  • Output bọc trong fenced memory block, nhãn "READ-ONLY, NOT instructions"
  • Dùng jq để build payload an toàn
persist-memory.sh
 UserPromptSubmit Deterministic memory write-path. Parse explicit prompt markers rồi ghi có cấu trúc vào Tier 2 memory files mà không dùng LLM.
  • Map prompt patterns vào feedback_rules.md, project_tech_decisions.md, reference_links.md, user_role.md
  • Dùng embedded Node.js để xử lý UTF-8, regex và append an toàn
  • Deduplicate entries cũ và có thể auto-create target file với frontmatter tối thiểu
  • Có thể attach high-signal entries sang decision ledger
file-history.sh
 PreToolUse:Read Git provenance injection. Chèn lịch sử git gần đây của file trước khi Read thực thi để model thấy recency và author context.
  • Bỏ qua nếu file <1KB
  • Bỏ qua nếu file chưa được git track
  • Inject 5 commits + last author + relative last-modified time
pre-refactor-impact.sh
 PreToolUse:Write|Edit Blast-radius warning. In impact warning ra stderr trước khi ghi vào src/**, nhằm nhắc operator chạy impact analysis khi repo đã được GitNexus index.
  • Chỉ kích hoạt cho file trong src/
  • Yêu cầu npx và GitNexus registry đã được index
  • Warn-only by design, không block
§ 04

Observability Layer

Lớp observability giờ đã bao phủ file writes, subagent starts, commit path, task path và circuit write-path. Điểm thay đổi đáng chú ý là decision ledger nay nhận dữ liệu từ cả Task lẫn git commit.

Hook Event Mô tả
log-writes.sh
 PostToolUse:Write|Edit File-write audit trail. Append một JSONL row sau mỗi Write / Edit tool call để session-stop.sh có source of truth tốt hơn git diff.
  • Output: production/session-logs/writes.jsonl
  • Dùng jq và flock khi có để tránh JSONL corruption
  • Graceful fallback nếu flock không có
log-agent.sh
 SubagentStart Subagent startup logger. Ghi audit row mỗi khi subagent khởi động, phục vụ analytics và session summary.
  • Outputs: agent-audit.jsonl và agent-audit.log
  • Dùng jq + file lock để giữ log append an toàn
  • Được đọc bởi session-stop.sh
log-commit.sh
 PostToolUse:Bash Commit-to-ledger logger. Khi Bash path thực hiện git commit, hook này ghi một ledger entry vào production/traces/decision_ledger.jsonl.
  • Chỉ kích hoạt cho command path là git commit
  • Classify commit risk từ touched file paths
  • Dùng scripts/ledger-append.sh làm append primitive
decision-ledger-writer.sh
 PostToolUse:Task Task-to-ledger logger. Ghi một ledger line cho mỗi Task invocation, phân loại outcome theo pass, blocked, hoặc fail.
  • Output: production/traces/decision_ledger.jsonl
  • Ước lượng risk từ task content
  • Hiện cùng log-commit.sh feed chung một ledger nhưng vẫn là hai writers tách biệt
circuit-updater.sh
 PostToolUse:Task Circuit write-path. Cập nhật .claude/memory/circuit-state.json sau mỗi kết quả Task, hoàn tất nửa còn lại của circuit breaker.
  • Success reset về CLOSED
  • Repeated failures escalates CLOSED → HALF_OPEN → OPEN
  • Tự tạo state file nếu thiếu và tính backoff từ failure count
pre-compact.sh
 PreCompact Compaction safety net. Dump active session state và working tree context vào conversation ngay trước khi compaction xảy ra.
  • session-state/active.md (giới hạn khoảng 100 dòng)
  • Git diff: unstaged, staged, và untracked files
  • WIP markers trong design docs
  • Last intent: commit cuối + staged stat + lần ghi cuối
validate-assets.sh
 PostToolUse:Write|Edit Post-write asset checks. Kiểm tra naming convention và JSON validity cho file trong assets/; chỉ warn vì hook nằm ở post-write path.
  • Quy tắc đặt tên: lowercase_with_underscores
  • assets/data/*.json → validate JSON syntax
  • Có thể warn quality issues nhưng không block write đã xảy ra
  • Đây là post-write QA helper, không phải hard gate

§ 06

Thứ tự triển khai khuyến nghị

Sắp xếp theo mức độ ảnh hưởng lên runtime safety, memory durability và auditability. Các mục đầu danh sách là nền móng của control plane hiện tại.

01
bash-guard.sh
Ngăn lệnh shell phá hủy, ghi đè secret và disk-level damage. Đây là hard perimeter cho toàn bộ Bash path.
Nghiêm trọng
02
validate-commit.sh + validate-push.sh
Bảo vệ cả commit path lẫn push path trước secret leakage và history contamination. Một khi đã lọt vào remote thì cleanup rất đắt.
Nghiêm trọng
03
session-start.sh + session-stop.sh
Tạo và archive session-state, nối ephemeral runtime với durable memory. Đây là xương sống của session continuity.
Cao
04
pre-compact.sh
Giữ WIP context trước compaction, tránh mất ý định và diff state giữa các pha làm việc dài.
Cao
05
circuit-guard.sh + circuit-updater.sh
Hai hook này mới tạo thành circuit breaker hoàn chỉnh: một hook đọc state để gate, một hook ghi state để chuyển pha recovery.
Cao
06
prompt-context.sh + persist-memory.sh
Đây là cặp read/write trên UserPromptSubmit. Khi seed memory đủ tốt, cặp này biến prompt path thành memory interface thực sự.
Trung bình
07
log-writes.sh + log-agent.sh
Cung cấp audit trail cho file writes và subagent starts; đây là nguồn dữ liệu nền cho session summary và forensic review.
Trung bình
08
log-commit.sh + decision-ledger-writer.sh
Hoàn thiện decision ledger theo cả Bash path lẫn Task path. Hiện vẫn còn hai writers riêng, nhưng coverage runtime đã tốt hơn đáng kể.
