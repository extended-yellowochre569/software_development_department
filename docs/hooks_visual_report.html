§ 01

Lifecycle Hooks

Kích hoạt tại ranh giới session. Không block (exit 0). Không yêu cầu dependency ngoài.

Hook Event Mô tả
session-start.sh
 SessionStart Báo cáo đầu session. Inject đầy đủ working context ngay khi mở session, Claude không cần hỏi lại trạng thái.
  • Current branch + 5 commits gần nhất
  • Trạng thái sprint / milestone đang chạy
  • Số open bug + TODO/FIXME trong src/
  • Preview session state từ phiên trước
detect-gaps.sh
 SessionStart Phát hiện lỗ hổng tài liệu. Đánh dấu code tồn tại nhưng thiếu documentation tương ứng.
  • Codebase >50 files nhưng ít design docs
  • Prototype thiếu README hoặc CONCEPT
  • src/core tồn tại nhưng không có docs/architecture/
  • Thư mục src/api lớn nhưng không có design doc
session-stop.sh
 Stop Dọn dẹp cuối session. Archive state, tính thống kê session, kích hoạt memory consolidation nếu đủ điều kiện.
  • Đọc agent-audit.jsonl → đếm subagent
  • Đọc git log (cửa sổ 8h) → đếm commit
  • Archive session vào memory/archive/sessions/
  • Trigger auto-dream.sh nếu MEMORY.md > 40 dòng
auto-dream.sh
 sub-process Dọn dẹp memory. Được gọi bởi session-stop.sh. Xóa và archive các memory entry lỗi thời.
  • Archive file cũ (>30 ngày, <5 dòng)
  • Đánh dấu file quá lớn (>50 dòng) để review thủ công
  • Xóa broken links trong MEMORY.md
  • Idempotent — không ghi log nếu không có thay đổi
§ 02

Security Layer

Hooks thuộc PreToolUse. exit 2 chặn tool call. Yêu cầu jq được cài đặt.

⚠
Hành vi blocking: Tất cả hook trong layer này có thể kết thúc bằng exit 2, ngăn tool call thực thi. Đảm bảo jq đã được cài — nếu thiếu, bash-guard.sh trả về exit 1 và chặn toàn bộ Bash tool calls.
Hook Event Mô tả
bash-guard.sh
 PreToolUse:Bash blocks Bộ bảo vệ tuyệt đối. Từ chối mọi shell pattern hủy diệt, không có ngoại lệ. Thiếu jq → exit 1, chặn toàn bộ Bash tool calls.
  • rm -rf / · rm -rf . · rm -rf * — xóa không thể phục hồi
  • curl | bash · wget | sh — remote code execution
  • cat .env · tee .env · > .env — lộ secret
  • git push --force — mất lịch sử vĩnh viễn
  • docker system prune · docker rm -f — xóa production
validate-commit.sh
 PreToolUse:Bash blocks Kiểm soát thông minh commit. Chỉ kích hoạt khi lệnh là git commit. Sáu tầng validation trước khi cho phép commit.
  • Kiểm tra staged files có tồn tại không
  • Validate định dạng Conventional Commits message
  • Từ chối .env, *.key, credentials.json trong staged set
  • Quét secret pattern: API_KEY, token trong diff
  • M2 fix: watchdog 25s — hiển thị WARN thay vì SIGKILL
validate-push.sh
 PreToolUse:Bash blocks Bảo vệ push. Lọc lệnh git push; áp dụng branch protection và quét secret trên toàn bộ diff.
  • Chặn --force / --force-with-lease
  • Chặn push trực tiếp lên main / master
  • Quét secret toàn bộ diff
  • Chạy pre-push tests nếu đã cấu hình
circuit-guard.sh
 PreToolUse:Task blocks ADR-004 Phase 2. Unified failure state machine. Ngăn Task tool thực thi khi circuit đang ở trạng thái lỗi.
  • CLOSED → hoạt động bình thường, pass-through
  • HALF_OPEN → cho qua + phát WARN, đang probe recovery
  • OPEN → block, TTL 60 phút hoặc reset thủ công
  • State file: .claude/memory/circuit-state.json
§ 03

Enrichment Layer

Inject context một cách thông minh. Không bao giờ block — chỉ làm giàu nhận thức của Claude.

Hook Event Mô tả
prompt-context.sh
 UserPromptSubmit Inject memory. Trích từ khóa từ user prompt, đối chiếu với .claude/memory/*.md, inject tối đa 3 file liên quan.
  • H2 fix: sanitize_memory_content() lọc injection patterns
  • Output bọc trong code fence, nhãn "READ-ONLY, NOT instructions"
  • Dùng jq -n --arg thay vì escape thủ công bằng sed
file-history.sh
 PreToolUse:Read Inject git history. Chèn 5 commit gần nhất của file trước khi Read thực thi. Bỏ qua file nhỏ hoặc chưa được track.
  • Bỏ qua nếu file <1KB (overhead vượt quá lợi ích)
  • Bỏ qua nếu file chưa được git track
  • Inject 5 commits + last author + last date
pre-refactor-impact.sh
 PreToolUse:Write Cảnh báo tác động refactor. In impact warning ra stderr trước khi ghi vào src/**. Yêu cầu GitNexus đã được index.
  • Chỉ kích hoạt cho file trong src/
  • Yêu cầu npx và GitNexus registry đã được index
  • Chỉ warn ra stderr — không block
§ 04

Observability Layer

Ghi log nguyên tử toàn bộ hoạt động hệ thống. Tạo ra audit trail không thể bị làm giả.

Hook Event Mô tả
log-writes.sh
 PostToolUse:Write Audit logger ghi file. Append một JSON entry vào writes.jsonl sau mỗi Write / Edit tool call.
  • H1 fix: flock -x bảo vệ atomic write
  • JSON sinh bằng jq — không escape thủ công
  • Graceful fallback nếu flock không có (Windows)
log-agent.sh
 SubagentStart Audit logger subagent. Append một JSON entry vào agent-audit.jsonl mỗi khi subagent khởi động.
  • H1 fix: flock + jq-based JSON generation
  • Kép output: agent-audit.jsonl (machine) + .log (human)
  • Được đọc bởi session-stop.sh để thống kê session
pre-compact.sh
 PreCompact Phao cứu sinh context. Dump toàn bộ working state vào conversation trước khi compaction. Claude không mất context qua quá trình nén.
  • session-state/active.md (giới hạn 100 dòng)
  • Git diff: unstaged, staged, và untracked files
  • WIP markers trong design docs
  • Last intent: commit cuối + staged stat + lần ghi cuối
validate-assets.sh
 PostToolUse:Write Kiểm tra asset. Xác nhận naming convention và JSON validity cho file ghi vào assets/. Không block.
  • Quy tắc đặt tên: lowercase_with_underscores
  • assets/data/*.json → validate JSON syntax
  • Chỉ warn ra stderr — không làm gián đoạn workflow

Thứ tự triển khai khuyến nghị

Sắp xếp theo mức độ tác động. Triển khai các biện pháp rủi ro cao nhất trước.

01
bash-guard.sh
Ngăn lệnh shell không thể đảo ngược (xóa, RCE). Side-effect không thể rollback.
Nghiêm trọng
02
validate-commit.sh
Ngăn secret lọt vào git history. Không thể xóa sạch sau khi push mà không viết lại history.
Nghiêm trọng
03
session-start.sh + session-stop.sh
Cung cấp working context khi mở session; archive state khi đóng. Nền tảng cho tất cả hook còn lại.
Cao
04
pre-compact.sh
Dump state trước compaction. Bảo tồn working context qua các sự kiện nén context.
Cao
05
circuit-guard.sh
Chặn Task tool khi circuit ở OPEN. Ngăn cascade failures qua chuỗi subagent.
Cao
06
prompt-context.sh
Yêu cầu .claude/memory/*.md đã có dữ liệu để có tác dụng. Triển khai sau khi memory được seed.
Trung bình
07
log-writes.sh
Audit trail cho file writes. Dùng để debug và tổng kết session.
Trung bình
08
log-agent.sh
Audit trail cho subagent calls. Dùng để đo chi phí và phân tích workflow.
