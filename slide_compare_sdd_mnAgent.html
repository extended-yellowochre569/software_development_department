  • Enterprise-ready: RBAC, identity management

Section 6 — Kịch Bản Áp Dụng
4 Kịch Bản Thực Tế
1
Solo Developer / Open Source
→ Dùng SDD
Free, full control, embedded knowledge. Không cần parallel execution. Audit trail từng bước là lợi thế, không phải nhược điểm.
2
Startup — Ship trong tuần
→ MNA + SDD Harness
Giữ: agent definitions, skills, governance. Bỏ: bash hooks, file-memory. Gain: autonomy, true parallel, built-in observability.
3
Enterprise — Regulated Industry
→ SDD + MNA Hybrid
SDD cho tasks cần approval (code review, deploy). Managed Agents cho background tasks (monitoring, reporting). Best of both worlds.
4
Platform / SaaS Builder
→ MNA infra + SDD template
MNA handles: multi-tenant isolation, billing, scaling. SDD provides: agent specialization patterns. Giống Notion, Asana, Sentry đang làm.

Section 7 — Migration Roadmap
SDD → Managed Agents: 3 Phases
Phase 01
Prepare
1–2 tuần
  • Export agent definitions sang format độc lập
  • Document tất cả workflow patterns
  • Map 12 path-scoped rules sang MNA permission model
  • Port memory structure sang cloud storage (Supabase/Redis)
Phase 02
Hybrid
2–4 tuần
  • Setup MNA project trong Claude Console
  • Migrate 3-5 agents critical (cto, lead-programmer, qa-lead)
  • Replace bash hooks → MNA event hooks
  • Build observability dashboard thay log-agent.sh
Phase 03
Full Migration
4–8 tuần
  • Migrate toàn bộ 27 agents
  • Implement self-evaluation loops cho QA
  • Enable true parallel (team-* skills)
  • MCP integrations: Jira, Slack, GitHub
Không cần thay đổi: Agent domain boundaries · Conflict resolution paths · Coding standards · Templates · Governance philosophy (ask → options → decide → draft → approve)
